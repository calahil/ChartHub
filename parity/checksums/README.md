# Parity Checksums

Committed checksum manifests live here.

Purpose:

- Record the expected signatures for non-transcoded outputs.
- Provide a stable, reviewable baseline independent of local binary artifacts.

Notes:

- Do not commit local output files from `.parity-artifacts/`.
- Commit only normalized checksum manifests generated by parity tooling.
- To refresh committed oracle checksums locally, run opt-in parity tests with `CH_PARITY_UPDATE_CHECKSUMS=1`.

Each checksum file entry contains:

- `path`: relative path inside the fixture output directory
- `sha256`: lowercase sha256 hex digest
- `role`: `audio`, `chart`, `metadata`, `art`, or `artifact`
- `comparison`: `byte` or `functional`
